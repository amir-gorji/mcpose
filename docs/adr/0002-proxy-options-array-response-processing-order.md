# ProxyOptions middleware arrays use response-processing order

The `toolMiddleware`, `resourceMiddleware`, and `listToolsMiddleware` arrays in `ProxyOptions` are in **response-processing order**: the first element processes the response first (innermost layer), the last element processes it last (outermost layer). This is the opposite of `compose()`, which takes outermost-first. We chose this order because the dominant use case is `[piiMW, auditMW]` — you want to say "PII runs before audit" meaning "PII redacts the response before audit ever sees it," which maps naturally to array position. If we had used outermost-first (matching `compose()`), that same guarantee would require writing `[auditMW, piiMW]`, which reads backwards relative to intent. The two conventions are explicitly not interchangeable; `ProxyOptions` arrays must not be passed directly to `compose()`.
